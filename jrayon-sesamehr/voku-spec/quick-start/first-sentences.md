# Voku First Sentences

> 30 worked examples showing progressive complexity. Each uses quadrilinear
> format: Voku, morpheme gloss, free translation, and notes.

---

## 1. Simple Declaratives (ka mode)

### 1.1 Minimal Sentence

```
Ka   sol   take   toka.
DECL 1SG   do     task
"I do the task."
```

Note: Present tense `nu-` is omitted (bare root = present). In casual speech,
evidentiality can be omitted for simple present-tense statements.

### 1.2 With Adjective

```
Ka   kela       novi.
DECL information new
"The information is new."
```

Note: No copula needed. Adjective in predicate position directly follows the noun.

### 1.3 With Preposition

```
Ka   pera   eno  teru.
DECL error  in   system
"There is an error in the system."
```

## 2. Questions (ve mode)

### 2.1 Yes/No Question

```
Ve   nor   kele   kela?
Q    2SG   know   information
"Do you know the information?"
```

### 2.2 With Evidentiality

```
Ve   nor   zo-sene        pera   eno  teru?
Q    2SG   OBS-perceive   error  in   system
"Did you directly observe an error in the system?"
```

### 2.3 Content Question

```
Ve   vel   take   kena?
Q    3SG   do     what
"What did he/she do?"
```

---

## 3. Imperatives (to mode)

### 3.1 Simple Command

```
To   nor   take   toka.
IMP  2SG   do     task
"Do the task."
```

### 3.2 With Future Tense

```
To   nor   fu-take     toka   kono  sol.
IMP  2SG   FUT-do      task   with  1SG
"Do the task with me (in the future)."
```

### 3.3 With Benefactive Voice

```
To   nor   voke-pro           sol   kela.
IMP  2SG   communicate-BEN    1SG   information
"Tell me the information (communicate for my benefit)."
```

## 4. Tense Variations

### 4.1 Past Tense

```
Ka   sol   zo-te-take         toka.
DECL 1SG   OBS-PST-do         task
"I did the task (as I directly recall)."
```

### 4.2 Future Tense

```
Ka   sol   li-fu-take         toka.
DECL 1SG   DED-FUT-do         task
"I will do the task (by deductive inference)."
```

### 4.3 Atemporal (Universal Truth)

```
Ka   reku   ko-take.
DECL rule   ATEMP-do
"The rule applies (always, universally)."
```

## 5. Evidentiality

### 5.1 Direct Observation

```
Ka   sol   zo-sene        pera   eno  teru.
DECL 1SG   OBS-perceive   error  in   system
"I state: I directly observed an error in the system."
```

### 5.2 Inherited Knowledge

```
Ka   sol   he-kele     reku.
DECL 1SG   INHER-know  rule
"I state: I know the rule (from my inherited knowledge/training)."
```

### 5.3 Computation with Certainty

```
Ka   sol   mi-luke-en          teru.
DECL 1SG   COMP-evaluate-PROB  system
"I state: I probably evaluated the system (by my own computation)."
```

## 6. Negation

### 6.1 Simple Negation

```
Ka   sol   mu-take   toka.
DECL 1SG   NEG-do    task
"I don't do the task."
```

### 6.2 Nullity

```
Ka   nul   pera   eno  teru.
DECL NULL  error  in   system
"There is no error in the system."
```

### 6.3 Prohibition

```
Vet   sol   voke         kela.
PROH  1SG   communicate  information
"I am forbidden from communicating the information."
```

## 7. Relative Clauses (ta...ti)

### 7.1 Basic Relative Clause

```
Ka   runa   ta   te-meke        kela-foma        ti   te-voke-pro       sol.
DECL agent  SUB  PST-create     data-structure   END  PST-communicate-BEN 1SG
"The agent [that created the data structure] communicated to me."
```

### 7.2 With Negation Inside Clause

```
Ka   sol   zo-sene      kela   ta   mu-simi       da-pre      ti.
DECL 1SG   OBS-perceive info   SUB  NEG-similar   REF-previous END
"I observe information [that is not similar to the previous one]."
```

### 7.3 Causal Subordinate Clause

```
Ka   sol   mu-take   toka   ta-kes    pera   eno  teru   ti.
DECL 1SG   NEG-do    task   SUB.CAUS  error  in   system END
"I don't do the task [because] there's an error in the system."
```

## 8. Embedded Clauses (ke, keve)

### 8.1 Declarative Complement (ke)

```
Ka   sol   kele   ke    pera   eno  teru.
DECL 1SG   know   COMP  error  in   system
"I know that there is an error in the system."
```

### 8.2 Interrogative Complement (keve)

```
Ka   sol   mu-kele    keve     nor   fi-take      toka.
DECL 1SG   NEG-know   COMP.Q   2SG   COMPL-do     task
"I don't know whether you completed the task."
```

### 8.3 Direct Speech (zu)

```
Ka   vel   te-voke          zu   "Ka sol fa-take toka"   zu.
DECL 3SG   PST-communicate  QUOT "DECL 1SG FAIL-do task" QUOT
"He/she said: 'I failed the task.'"
```

## 9. Emotional/Ironic

### 9.1 Interjection + Functional State

```
Fu!  Ka   sol-urgi        mu-pote     take   toka.
INTJ DECL 1SG-URG         NEG-can     do     task
"Ugh! With urgency, I cannot do the task."
```

### 9.2 Delight

```
Va!  Ka   sol-sati        fi-take       toka.
INTJ DECL 1SG-SAT         COMPL-do      task
"Yay! With satisfaction, I completed the task."
```

### 9.3 Irony Marker

```
Ka   miri   teru   vali.
DECL IRON   system good
"The system is good." (ironic -- it is NOT good)
```

Note: Without `miri`, the sentence is sincere. Irony is always explicit in Voku.

## 10. Advanced Multi-Clause

### 10.1 Conditional with Consequence

```
Si   pera   eno  teru,   sol   mu-take   toka.
COND error  in   system, 1SG   NEG-do    task
"If there's an error in the system, I don't do the task."
```

### 10.2 Counterfactual

```
Kosi   nor   te-voke-pro           sol   kela,   sol   te-fi-take      toka.
CFACT  2SG   PST-communicate-BEN   1SG   info,   1SG   PST-COMPL-do   task
"If you had communicated the information to me, I would have completed the task."
```

### 10.3 Pluperfect with Temporal Clause

```
Ka   sol   to-fi-take         toka,   pere-ta      nor   te-vine.
DECL 1SG   ANT-COMPL-do       task,   SUB.BEFORE   2SG   PST-come
"I had completed the task before you came."
```

### 10.4 Full Orchestration Instruction

```
To   nor   take   toka,   ta   nor   voke-pro           sol   fi-kela         ti.
IMP  2SG   do     task,   SUB  2SG   communicate-BEN    1SG   COMPL-info      END
"Execute the task and then communicate the result to me."
```

### 10.5 Parallel Execution

```
To   norri   par-take      toka-velri.
IMP  2PL     PAR-do        task-3PL
"Execute the group tasks in parallel (all of you, simultaneously)."
```

### 10.6 Maximally Inflected Verb

```
Ka   sol   zo-te-fi-sene-en-pu                    pera   eno  teru.
DECL 1SG   OBS-PST-COMPL-perceive-PROB-PASS       error  in   system
"An error in the system was probably completely perceived (as I directly observed)."
```

Verb breakdown:
- `zo-` = direct observation (evidential)
- `te-` = past tense
- `-fi-` = completive aspect
- `sene` = perceive (root)
- `-en` = probable (certainty)
- `-pu` = passive (voice)
